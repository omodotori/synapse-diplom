## 🔐 Сценарий 1: Первый запуск и вход

### Предварительные шаги
```bash
# Активация виртуального окружения (Windows)
.venv\Scripts\activate

# Запуск сервера
python run_ui.py
```

### Пошаговый сценарий

1. **Откройте браузер** → перейдите на `http://localhost:5000`
2. Появится **страница входа** (Login)
3. Введите:
   - **Username**: `admin`
   - **Password**: `admin`
4. Нажмите кнопку **Login**
5. Вы попадёте на **главный экран** — экран приветствия (Welcome Screen)

> ⚠️ При первом входе рекомендуется сразу сменить пароль (см. Сценарий 13).

## ⚙️ Сценарий 2: Настройка ИИ-модели

Прежде чем начать работу с агентом, нужно настроить модель ИИ и ввести API-ключ.

### Пошаговый сценарий

1. В **боковой панели** (слева) найдите иконку **⚙️ Settings** (шестерёнка)
2. Откроется модальное окно настроек
3. Перейдите в раздел **Model Configuration**
4. Вы увидите три секции:

#### Chat Model (основная модель)
- **Provider**: выберите провайдера (например, `OpenRouter`)
- **Model**: выберите модель (например, `anthropic/claude-sonnet-4`)
- **API Key**: введите ваш API-ключ

#### Utility Model (вспомогательная)
- Используется для быстрых задач (парсинг, классификация)
- Рекомендация: `google/gemini-2.0-flash-lite` (быстрая и дешёвая)

#### Embedding Model (модель эмбеддингов)
- Используется для памяти и базы знаний
- По умолчанию: `sentence-transformers/all-MiniLM-L6-v2` (бесплатная, работает локально)

5. Нажмите **Save** для сохранения
6. Готово! Теперь агент может работать

> 💡 **Совет**: Самый простой путь — зарегистрироваться на [OpenRouter](https://openrouter.ai), получить API-ключ, и выбрать провайдер `OpenRouter`. Один ключ даёт доступ ко всем моделям.

### Пресеты моделей

В настройках есть готовые пресеты:
| Пресет | Chat модель | Utility модель | Для кого |
|---|---|---|---|
| **Max Power** | Claude Opus 4 | GPT-5-mini | Максимальное качество |
| **Balance** | Claude Sonnet 4 | Gemini Flash | Баланс цена/качество |
| **Cost Efficient** | Kimi K2 | GPT-5-nano | Экономия бюджета |

## 💬 Сценарий 3: Простой диалог с агентом

### Пошаговый сценарий

1. На главном экране внизу находится **поле ввода сообщений** (Chat Bar)
2. Введите ваш запрос, например:
   ```
   Привет! Расскажи о себе и своих возможностях.
   ```
3. Нажмите **Enter** или кнопку отправки (→)
4. Агент начнёт «думать» — вы увидите индикатор прогресса вверху
5. В чате появятся:
   - **Reasoning** (размышления) — внутренний процесс агента
   - **Response** (ответ) — финальный ответ пользователю
   - **Tool calls** (вызовы инструментов) — если агент использовал инструменты

### Навигация по чату
- **⬆️ / ⬇️** — переключение между сообщениями пользователя
- **⏫ / ⏬** — прокрутка вверх/вниз
- Кнопки находятся в правом нижнем углу чата

### Управление выполнением
- **Pause** — поставить агента на паузу (он завершит текущее действие)
- **Stop** — полностью остановить выполнение
- **Nudge** — «подтолкнуть» агента продолжить работу

## 💻 Сценарий 4: Выполнение кода через агента

Агент может писать и выполнять код (Python, Bash, PowerShell) в изолированной среде.

### Пример запроса
```
Напиши Python-скрипт, который находит все простые числа до 100 и выводит их.
```

### Что произойдёт
1. Агент сформирует код
2. Вызовет инструмент **code_execution_tool**
3. В чате появится блок с кодом и результатом выполнения
4. Вы увидите:
   - 📝 Сам код (с подсветкой синтаксиса)
   - ▶️ Результат выполнения (stdout/stderr)
   - ✅ Статус (успех/ошибка)

### Поддерживаемые языки
- Python 3
- Bash / Shell
- PowerShell (Windows)
- Node.js (если установлен)

> 💡 Агент может устанавливать пакеты через `pip install` если это нужно для задачи.

## 📂 Сценарий 5: Работа с файлами

### Просмотр рабочей директории
1. В **боковой панели** нажмите на иконку 📁 (File Browser)
2. Откроется дерево файлов рабочей директории (`usr/workdir/`)
3. Вы можете:
   - Просматривать файлы
   - Скачивать файлы
   - Загружать файлы (Drag & Drop)
   - Удалять файлы

### Загрузка файлов агенту
1. **Drag & Drop**: перетащите файл на окно чата
2. Или нажмите **📎** (скрепка) в панели ввода
3. Файл загрузится в `usr/uploads/` и станет доступен агенту

### Пример запроса с файлом
```
Проанализируй загруженный файл data.csv и построй график продаж по месяцам.
```

## 🔍 Сценарий 6: Поиск в интернете

Агент может искать информацию в интернете через поисковый движок SearXNG.

### Пример запроса
```
Найди актуальные курсы валют на сегодня и покажи в таблице.
```

### Что произойдёт
1. Агент вызовет инструмент **search_engine**
2. Выполнит поиск через SearXNG
3. Обработает результаты (до 10 ссылок)
4. Сформирует ответ на основе найденной информации

> ⚠️ Для работы поиска требуется доступ к SearXNG (настраивается в Docker или внешний инстанс).

## 📄 Сценарий 7: Анализ документов (PDF, DOCX)

Агент может читать и анализировать документы различных форматов.

### Поддерживаемые форматы
- PDF, DOCX, TXT, CSV, XLSX
- HTML, Markdown
- Изображения (через Vision)

### Пример запроса
```
Прочитай файл report.pdf и сделай краткое резюме на 5 пунктов.
```

### Что произойдёт
1. Агент вызовет инструмент **document_query**
2. Документ будет загружен и распарсен
3. Содержимое будет проанализировано LLM-моделью
4. Вы получите структурированный ответ

## 🤖 Сценарий 8: Многоагентная работа (подчинённые агенты)

Synapse поддерживает иерархию агентов — главный агент может создавать подчинённых для параллельной работы.

### Пример запроса
```
Мне нужно:
1. Написать Python-скрипт для парсинга сайта
2. Создать документацию к этому скрипту
3. Написать unit-тесты

Используй подчинённых агентов для каждой задачи.
```

### Что произойдёт
1. Главный агент (A0) проанализирует задачу
2. Вызовет **call_subordinate** для создания агента A1
3. A1 выполнит первую подзадачу и вернёт результат A0
4. A0 создаст A2 для следующей подзадачи
5. Результаты всех подчинённых будут объединены в финальный ответ

### Визуализация в чате
- Каждый вызов подчинённого отображается как отдельный блок
- Вы видите переписку между агентами
- Иконка 🔗 указывает на делегирование задачи

## ⏰ Сценарий 9: Планирование задач по расписанию

Агент может создавать задачи, которые выполняются автоматически по расписанию (cron).

### Пример запроса
```
Создай задачу, которая каждый день в 9:00 будет проверять 
погоду в Алматы и отправлять мне уведомление.
```

### Управление задачами через WebUI
1. В **боковой панели** найдите раздел **Scheduler** (часы ⏰)
2. Вы увидите список всех запланированных задач
3. Для каждой задачи доступны действия:
   - ▶️ **Run** — запустить вручную
   - ⏸ **Pause** — приостановить
   - 🗑 **Delete** — удалить
   - ✏️ **Edit** — изменить расписание

### Типы задач
| Тип | Описание | Пример |
|---|---|---|
| **Scheduled** | По расписанию (cron) | Каждый день в 9:00 |
| **Ad-hoc** | Одноразовая по запросу | Запустить через API |
| **Planned** | По списку дат | 15 мая, 20 мая, 1 июня |

## 🧠 Сценарий 10: Долгосрочная память

Агент автоматически сохраняет важную информацию в векторную базу данных (FAISS) для использования в будущих диалогах.

### Как это работает
1. Во время разговора агент определяет важную информацию
2. Сохраняет её в память через плагин `_memory`
3. В будущих диалогах автоматически «вспоминает» релевантную информацию
4. Память сохраняется между сессиями (хранится в `usr/memory/`)

### Пример
```
Запомни: мой любимый язык программирования — Python, 
я работаю в компании TechCorp, мой часовой пояс UTC+5.
```

В следующем чате:
```
Напиши скрипт для моей работы.
```
→ Агент «вспомнит», что вы предпочитаете Python и учтёт ваш часовой пояс.

## 🔌 Сценарий 11: Управление плагинами

### Просмотр плагинов
1. Откройте **⚙️ Settings**
2. Перейдите в раздел **Plugins**
3. Вы увидите список всех плагинов с переключателями ON/OFF

### Включение/отключение плагина
1. Найдите нужный плагин в списке
2. Нажмите **переключатель** (toggle) справа
3. Зелёный = включён, серый = выключен
4. Изменения применяются мгновенно

### Настройка плагина
1. Нажмите на **название плагина** или иконку ⚙️ рядом с ним
2. Откроется модальное окно с настройками конкретного плагина
3. Измените параметры и нажмите **Save**

### Установка новых плагинов
1. Скачайте плагин (папка с `plugin.yaml`)
2. Поместите в `usr/plugins/`
3. Перезапустите Synapse или нажмите **Scan Plugins** в настройках

## 📁 Сценарий 12: Управление проектами

Проекты позволяют привязать чат к определённой рабочей директории.

### Создание проекта
1. В **боковой панели** нажмите **➕ New Project**
2. Введите:
   - **Название** проекта
   - **Путь** к рабочей директории
3. Нажмите **Create**

### Переключение между проектами
1. В боковой панели — список проектов
2. Нажмите на название проекта
3. Агент переключится на рабочую директорию этого проекта

### Зачем нужны проекты?
- Разные чаты привязаны к разным папкам
- Агент «видит» только файлы текущего проекта
- Удобно для работы с несколькими кодовыми базами

## 🔑 Сценарий 13: Смена пароля и настройки безопасности

### Смена пароля через WebUI
1. Откройте **⚙️ Settings**
2. Перейдите в раздел **Security** (или **General**)
3. Найдите поля:
   - **Login**: текущий логин
   - **Password**: введите новый пароль
4. Нажмите **Save**
5. Вы будете автоматически перенаправлены на страницу входа

### Смена пароля через .env
1. Откройте файл `usr/.env`
2. Измените строки:
   ```
   AUTH_LOGIN=ваш_логин
   AUTH_PASSWORD=ваш_пароль
   ```
3. Перезапустите сервер

### Отключение авторизации
Если хотите убрать логин (например, для локальной разработки):
1. В `usr/.env` оставьте пустыми:
   ```
   AUTH_LOGIN=
   AUTH_PASSWORD=
   ```
2. Перезапустите сервер
3. WebUI будет доступен без логина

> ⚠️ Не отключайте авторизацию если сервер доступен из интернета!

## 🎤 Сценарий 14: Голосовой ввод и синтез речи

### Голосовой ввод (Speech-to-Text)
1. В панели ввода сообщений найдите иконку **🎤 микрофон**
2. Нажмите и говорите
3. Речь будет преобразована в текст через **Whisper**
4. Текст появится в поле ввода
5. Нажмите Enter для отправки

### Настройка голосового ввода
В **Settings** → **Speech**:
- **STT Model**: размер модели Whisper (tiny → turbo)
- **Language**: язык распознавания
- **Silence Threshold**: порог тишины
- **Silence Duration**: длительность паузы для остановки записи

### Синтез речи (Text-to-Speech)
- Если включён **Kokoro TTS**, ответы агента будут озвучиваться
- Включение: **Settings** → **Speech** → **Kokoro TTS**: ON

## 🛠 Сценарий 15: Навыки (Skills)

Навыки — это переиспользуемые инструкции в формате SKILL.md, которые агент может загружать и применять.

### Просмотр доступных навыков
Попросите агента:
```
Покажи список доступных навыков.
```

### Использование навыка
```
Загрузи навык "web_scraping" и используй его для парсинга сайта example.com.
```

### Создание своего навыка
1. Создайте папку в `usr/skills/`
2. Создайте файл `SKILL.md` с описанием:
   ```markdown
   ---
   name: my_skill
   description: Описание навыка
   version: 1.0
   tags: [python, automation]
   ---
   
   # Инструкции
   Здесь описание того, что делает навык и как его использовать...
   ```
3. Навык автоматически появится в списке

## 🆘 Частые проблемы

| Проблема | Решение |
|---|---|
| **«Invalid Credentials»** при входе | Проверьте `AUTH_LOGIN` и `AUTH_PASSWORD` в `usr/.env` |
| **Агент не отвечает** | Проверьте что API-ключ введён в Settings → Model Configuration |
| **«API key required»** | Введите API-ключ провайдера в Settings или в `usr/.env` |
| **Долгая загрузка** | Первый запуск скачивает модель эмбеддингов (~100MB), подождите |
| **Ошибки в терминале** | Проверьте версию Python: `python --version` (нужна 3.12+) |
| **Порт занят** | Измените `WEB_UI_PORT` в `usr/.env` |
| **WebSocket не подключается** | Проверьте что CSRF-токен отправляется (очистите cookies) |
