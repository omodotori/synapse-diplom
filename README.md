## 📝 2. Краткое описание

**Synapse** — это мощный автономный агентный ИИ-фреймворк с открытым исходным кодом. 
В отличие от обычных чат-ботов, Synapse **использует вашу операционную систему как инструмент**. Он может писать и запускать код в изолированной среде, искать информацию в браузере, работать с файловой системой, а также делегировать подзадачи другим "подагентам". 

Проект идеально подходит для разработчиков, исследователей и бизнеса, кому нужен корпоративный ИИ-ассистент с поддержкой 100+ LLM (через LiteLLM), долговременной памятью (RAG) и интеграцией с мессенджерами (Telegram, WhatsApp).

## ✨ 4. Возможности (Features)

*   **💻 Code Execution:** Выполняет Python, Bash, Node.js и PowerShell код в изолированной Docker-песочнице.
*   **👥 Многоагентная архитектура:** Главный агент может создавать специализированных "подчиненных" агентов для параллельного решения сложных задач.
*   **🌐 Управление браузером (Playwright):** Навигация по сайтам, клики, парсинг данных и создание скриншотов.
*   **🔍 Search Engine:** Приватный поиск через встроенный SearXNG.
*   **🧠 Долгосрочная память (RAG):** Векторная база данных на FAISS для сохранения контекста между сессиями.
*   **⏰ Планировщик (Scheduler):** Запуск задач по cron-расписанию или через таймер.
*   **📱 Встроенные боты (Telegram & WhatsApp):** Управление агентом прямо с вашего телефона через мессенджеры (включая отправку фото и аудио).
*   **🔌 Плагинная система:** Поддержка хуков для легкого расширения функционала без изменения ядра.

├── plugins/              # 🧩 Системные плагины (Telegram, WhatsApp, Email, Автозапуск)
├── prompts/              # 📝 Системные промпты агента
├── knowledge/            # 📚 База знаний агента (внутренние справки и скиллы)
│
├── webui/                # 🖥 Фронтенд на Alpine.js
│   ├── index.html        # Главная страница
│   ├── components/       # UI компоненты (сайдбар, модальные окна, настройки)

│
├── usr/                  # 👤 Пользовательская директория (НЕ трекается в git)

│

## ⚡ 8. Установка (Installation)

Самый безопасный способ развертывания — **Docker Compose**. 

### 1. Клонирование и настройка
```bash
git clone https://github.com/your-username/synapse.git
cd synapse

# Создаем файл конфигурации на основе примера
cp .env.example usr/.env
```
Откройте `usr/.env` и впишите API-ключ нужного провайдера (подробнее в разделе 9).

### 2. Запуск через Docker (Рекомендуется)
```bash
cd docker/run
docker-compose up -d --build
```
После успешного запуска интерфейс будет доступен по адресу: 👉 **http://localhost:50080**

### 3. Локальный запуск (Для разработчиков)
```bash
# Создание виртуального окружения
python -m venv .venv
source .venv/bin/activate  # Для Windows: .venv\Scripts\activate

# Установка зависимостей (важно соблюдать порядок)
pip install -r requirements.txt
pip install -r requirements2.txt

# Запуск в Windows (скрипт откроет 2 терминала: Sandbox и Main сервер)
.\start.bat
```

## ⚙️ 9. Конфигурация (.env)

Файл `usr/.env` управляет всеми настройками.

| Переменная | Описание | По умолчанию |
|---|---|---|
| `WEB_UI_PORT` | Порт, на котором работает интерфейс | `50001` |
| `WEB_UI_HOST` | Хост (`127.0.0.1` или `0.0.0.0`) | `127.0.0.1` |
| `AUTH_LOGIN` | Логин для входа в WebUI | `admin` |
| `AUTH_PASSWORD` | Пароль для входа в WebUI (Обязательно смените!) | `admin` |
| `A0_SET_AGENT_PROFILE` | Загружаемый профиль агента | `synapse` |
| `API_KEY_OPENROUTER` | API-Ключ OpenRouter (рекомендуется) | `[пусто]` |
| `API_KEY_OPENAI` | API-Ключ OpenAI (GPT-4) | `[пусто]` |
| `API_KEY_ANTHROPIC` | API-Ключ Anthropic (Claude 3.5) | `[пусто]` |

*Вам достаточно заполнить ключ только от одного провайдера.*

## 🎮 10. Использование (Usage)

1. **Вход в систему:** Перейдите по адресу `http://localhost:50080` и авторизуйтесь (по умолчанию `admin` / `admin`).
2. **Настройка ИИ:** Зайдите в Настройки (⚙️) -> **Модели**, выберите желаемого провайдера и модель (например, Claude 3.5 Sonnet).
3. **Общение:** Просто напишите агенту задачу в чат. Например:
   > *"Создай папку `test_project` в моей рабочей директории, инициализируй там git и напиши простой python-скрипт для парсинга новостей."*
4. **Интеграция Telegram:** 
   * Перейдите в раздел "Плагины".
   * Включите **Telegram Integration**, нажмите шестеренку (⚙️).
   * Введите Bot Token от `@BotFather`.
   * Напишите боту `/start` с телефона!

## 📡 11. API документация

Synapse общается между фронтендом и бэкендом в основном через **WebSocket (Socket.IO)** и несколько REST эндпоинтов.

**Ключевые эндпоинты:**
*   `POST /api/login` — Авторизация (возвращает сессию и CSRF токен).
*   `GET /api/ping` — Проверка статуса сервера.
*   `WS /ws` — Основной канал связи Socket.IO.
    *   *Event:* `chat_message` — отправка сообщения агенту.
    *   *Event:* `agent_reply` — стриминг ответа от агента.
    *   *Event:* `monologue_start` / `monologue_end` — статусы "размышления" агента.

*Подробная документация по созданию плагинов и API доступна в `docs/agents/AGENTS.plugins.md`.*

## 🚀 14. Деплой (Production)

Для выкладки на продакшн сервер (VPS/VDS):
1. Обязательно измените `AUTH_PASSWORD` и установите надежный `ROOT_PASSWORD` в `usr/.env`.
2. Установите `WEB_UI_HOST=0.0.0.0`.
3. Используйте Docker Compose:
   ```bash
   cd docker/run
   docker-compose up -d --build
   ```
4. Рекомендуется настроить Nginx в качестве Reverse Proxy (с SSL-сертификатом от Let's Encrypt), проксируя трафик на внутренний порт контейнера (50080) с поддержкой WebSocket-заголовков (`Upgrade`, `Connection`).

## ❓ 18. FAQ / Известные проблемы

**В: Агент выдает ошибку "Permission denied" при попытке установить системный пакет (apt-get).**
**О:** В целях безопасности агент работает в Docker-контейнере от лица обычного пользователя (`synapse`). Вы можете повысить права агенту (или запустить его с профилем `hacker`), но мы рекомендуем ставить зависимости локально в `workdir`.

**В: WhatsApp мост не стартует.**
**О:** Убедитесь, что на сервере установлен `Node.js` v18+. Фреймворк попытается скачать зависимости моста автоматически, но для этого нужен `npm`.

**В: Не работает микрофон в веб-интерфейсе.**
**О:** Браузеры блокируют доступ к микрофону, если сайт загружен по `http://`. Для локальной разработки используйте `http://localhost` (он в исключениях), а для продакшна обязательно настройте `https://`.
