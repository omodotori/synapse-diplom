# Сценарий демонстрации проекта: Synapse 🧠

## 1. Краткое описание проекта
**Synapse** — это автономный агентный ИИ-фреймворк, который позволяет развернуть собственного мощного агента, способного использовать операционную систему как инструмент. В отличие от обычных чат-ботов, Synapse может писать и выполнять код, искать информацию в интернете, управлять файлами и делегировать задачи другим агентам.

## 2. Что умеет система
- [x] Исполнять код (Python, Bash, Node.js) в безопасной Docker-песочнице.
- [x] Создавать "подагентов" для параллельного выполнения задач.
- [x] Управлять браузером (через Playwright) для парсинга и взаимодействия с веб-страницами.
- [x] Работать с долгосрочной памятью (FAISS) — запоминать контекст между сессиями.
- [x] Интегрироваться с внешними мессенджерами (Telegram, WhatsApp) для удаленного доступа.
- [x] Поддерживать 100+ LLM моделей через LiteLLM (включая локальные через Ollama).

## 3. Пошаговый сценарий проверки (Демо-Флоу)

### Шаг 1: Инициализация и запуск
- Запускаем контейнер через `docker-compose up -d --build`.
- Открываем WebUI по адресу `http://localhost:50080`.
- Вводим дефолтные логин/пароль (`admin` / `admin`).

### Шаг 2: Базовое взаимодействие
- Заходим в настройки (⚙️), показываем панель выбора ИИ-провайдера.
- Пишем в чат простую задачу: *"Напиши python-скрипт, который выводит 'Hello World', и запусти его"*.
- Демонстрируем, как агент открывает внутренний терминал, пишет скрипт в `workdir` и присылает результат выполнения.

### Шаг 3: Демонстрация модулей (Браузер / Поиск)
- Просим агента: *"Зайди в интернет, найди текущий курс биткоина и создай текстовый файл `btc.txt` с результатом"*.
- Показываем процесс: агент использует `Search Engine` или `Browser`, парсит данные и использует `Code Execution` для создания файла.

### Шаг 4: Удаленный доступ (Telegram/WhatsApp)
- Открываем левое меню, переходим в список плагинов.
- Показываем включенный плагин **Telegram Integration**.
- Отправляем с телефона сообщение боту `/start` (или голосом) и получаем ответ от агента в мессенджер прямо во время презентации.

## 4. Какие модули показывать
- **WebUI (Alpine.js):** Интерфейс, настройки, логика чатов.
- **Инструменты (Tools):** `_code_execution`, `browser.py`, `knowledge_tool.py`.
- **Плагины (Plugins):** Telegram Integration, WhatsApp Integration.
- **Docker-среда:** Изолированная песочница, где агент выполняет код.

## 6. Как работает процесс тестирования
- Тестирование покрывает ядро агента, инструменты и авторизацию (CSRF, Flask Sessions).
- Интеграционные тесты (Pytest) проверяют корректную инициализацию плагинов и доступ к инструментам.
- Использование мокирования (Mocking) API-вызовов LLM-моделей позволяет проверять логику "размышления" агента без реальной траты токенов.

## 7. Как используется AI в проекте
- **Monologue Loop (Цикл размышлений):** AI получает задачу, анализирует её, выбирает нужный `Tool` (Инструмент), выполняет его и анализирует результат. Если есть ошибка — ИИ сам пытается её исправить.
- **RAG (Retrieval-Augmented Generation):** ИИ использует векторные эмбеддинги для поиска нужной информации по своей базе знаний и прошлым диалогам.
- **Multi-Agent (Делегирование):** Главный AI может инициировать запуск других AI с другим системным промптом (например, вызвать "Программиста", пока сам играет роль "Менеджера").
